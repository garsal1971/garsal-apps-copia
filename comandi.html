#000
Cerca la stored procedure KCBPANAG nel catalogo DB2
Esegui questa query per verificare che la stored procedure esista e in quale schema si trova. Se non appare nessuna riga, la proc non esiste o non hai i permessi per vederla. Comunicare il risultato (NAME, SCHEMA, PARM_COUNT).
🔍 diagnostica SQL DB2 z/OS 
SELECT NAME,
       SCHEMA,
       PARM_COUNT,
       LANGUAGE,
       SPECIFICNAME
FROM   SYSIBM.SYSROUTINES
WHERE  NAME = 'KCBPANAG'
📍 Dove: DBeaver → apri Script → incolla → esegui (F5) · Comunicare il risultato completo
#001
Leggi la firma completa dei parametri di KCBPANAG
Esegui dopo il task #000. Sostituisci DB2C con lo schema trovato nella query precedente. Mostra nome, tipo, lunghezza e direzione (P = IN, O = OUT) di ogni parametro. Questo permette di costruire la CALL corretta.
🔍 diagnostica SQL DB2 z/OS 
SELECT PARMNAME,
       ROWTYPE,
       TYPENAME,
       LENGTH,
       SCALE,
       ORDINAL
FROM   SYSIBM.SYSPARMS
WHERE  SCHEMA = 'DB2C'              -- <-- sostituire con lo schema trovato in #000
  AND  SPECIFICNAME IN (
           SELECT SPECIFICNAME
           FROM   SYSIBM.SYSROUTINES
           WHERE  SCHEMA = 'DB2C'   -- <-- idem
             AND  NAME   = 'KCBPANAG'
       )
ORDER  BY ORDINAL
📍 Dove: DBeaver → apri Script → incolla → esegui (F5) · Eseguire dopo #000 · Sostituire DB2C con lo schema reale

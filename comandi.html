#001
SQL — aggiorna la constraint history_action_check
La constraint su ts_history.action non include 'completed_late' (task completati in ritardo) né 'rollback' (storno punti). Ogni completamento di task late genera errore 23514.
Il fix è già nella migration 20260316130000 che verrà applicata automaticamente dal workflow, ma nel frattempo applica subito da SQL Editor.
🔴 urgente SQL Supabase SQL Editor 
ALTER TABLE ts_history
  DROP CONSTRAINT IF EXISTS history_action_check;

ALTER TABLE ts_history
  ADD CONSTRAINT history_action_check
    CHECK (action IN (
      'completed',
      'completed_late',
      'failed',
      'terminated',
      'archived',
      'skipped',
      'reactivated',
      'rollback'
    ));

#003
SQL — verifica completamento task via Telegram
Dopo aver cliccato "✅ Fatto" su un task in Telegram, verifica che:
• il task abbia status = 'terminated'
• la regola di notifica sia stata eliminata da cm_notification_rules
• la coda abbia status = 'completed'
⏳ test finale SQL 
-- Ultimi task completati (ultimi 30 minuti)
SELECT id, title, type, status, last_completed_date
FROM ts_tasks
WHERE status IN ('terminated', 'completed')
  AND last_completed_date >= now() - interval '30 minutes'
ORDER BY last_completed_date DESC
LIMIT 10;

-- Ultimi record in ts_history (verifica action)
SELECT task_id, from_status, to_status, action, points, timestamp
FROM ts_history
ORDER BY timestamp DESC
LIMIT 10;

Verifica policy RLS su cm_notification_rules e cm_notification_queue
Se le tabelle non hanno policy, l'utente autenticato non può leggere/scrivere. Esegui la query di verifica e, se mancano righe, esegui i CREATE POLICY.
SQL PROD 
-- Verifica policy esistenti
SELECT tablename, policyname
FROM pg_policies
WHERE tablename IN ('cm_notification_rules', 'cm_notification_queue');

-- Se NON ci sono righe per una tabella, esegui:

-- ALTER TABLE cm_notification_rules ENABLE ROW LEVEL SECURITY;
-- CREATE POLICY "utente autenticato" ON cm_notification_rules
--   FOR ALL USING (auth.role() = 'authenticated');

-- ALTER TABLE cm_notification_queue ENABLE ROW LEVEL SECURITY;
-- CREATE POLICY "utente autenticato" ON cm_notification_queue
--   FOR ALL USING (auth.role() = 'authenticated');

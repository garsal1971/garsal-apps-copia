#001
SQL — pulizia entry pending task con vecchio occurrence_id
I task avevano occurrence_id = "{rule_id}" (solo UUID, senza data/ora).
Il nuovo formato è "{rule_id}:{YYYY-MM-DD}:{HH:MM}", uniforme con gli habit.
Questo SQL elimina le entry pending/snoozed dei task nel vecchio formato per evitare orfane. La funzione fill-notification-queue le rigenererà con il formato corretto al prossimo run (#003).
⏳ da fare SQL Supabase 1° — eseguire per primo 
-- Elimina entry pending/snoozed dei task con occurrence_id nel vecchio formato
-- (vecchio formato: solo UUID senza ':' — es. "abc123-def456-...")
-- (nuovo formato:   "abc123-...:2026-03-15:00:00")
DELETE FROM cm_notification_queue
WHERE status IN ('pending', 'snoozed')
  AND app = 'tasks'
  AND occurrence_id IS NOT NULL
  AND occurrence_id NOT LIKE '%:%';

-- Verifica: deve restituire 0 righe dopo la pulizia
SELECT COUNT(*) AS vecchio_formato_rimasto
FROM cm_notification_queue
WHERE app = 'tasks'
  AND occurrence_id IS NOT NULL
  AND occurrence_id NOT LIKE '%:%';

  app,
  occurrence_id,
  fire_at AT TIME ZONE 'Europe/Rome' AS fire_at_rome,
  title,
  body,
  status
FROM cm_notification_queue
WHERE status = 'pending'
ORDER BY fire_at
LIMIT 20;

#001
SQL — pulizia stati sending e snoozed dismessi
Gli stati sending e snoozed non vengono più usati dopo il refactoring.
• sending — eliminato: nessun retry, status passa direttamente pending → sent/failed
• snoozed — eliminato: sostituito da INSERT di un nuovo row pending con nuovo fire_at
Row recenti (fire_at < 1 ora fa) → riportati a pending per la nuova finestra ±5 min.
Row più vecchi → marcati failed.
⏳ da fare SQL Supabase 1° — eseguire per primo 
-- Riporta a 'pending' i row recenti con stati dismessi
UPDATE cm_notification_queue
SET status = 'pending'
WHERE status IN ('sending', 'snoozed')
  AND fire_at >= now() - interval '1 hour';

-- Marca come 'failed' i row più vecchi con stati dismessi
UPDATE cm_notification_queue
SET status = 'failed'
WHERE status IN ('sending', 'snoozed')
  AND fire_at < now() - interval '1 hour';

-- Verifica: devono risultare 0 righe con stati dismessi
SELECT COUNT(*), status
FROM cm_notification_queue
WHERE status IN ('sending', 'snoozed')
GROUP BY status;

FROM cm_notification_queue

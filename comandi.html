Il webhook Telegram usa service_role per chiamare la RPC task_complete.
La funzione aveva solo GRANT EXECUTE TO authenticated: se Supabase ha revocato il PUBLIC EXECUTE, service_role non riesce a invocarla e il bottone "✅ Fatto" fallisce.
Questa migration aggiunge GRANT EXECUTE TO service_role per task_complete e habit_post_completion.

-- Verifica: le funzioni devono essere visibili al ruolo service_role
SELECT routine_name, grantee, privilege_type
FROM information_schema.routine_privileges
WHERE routine_name IN ('task_complete', 'habit_post_completion')
  AND grantee IN ('authenticated', 'service_role', 'PUBLIC')
ORDER BY routine_name, grantee;

#004
SQL — verifica che il task singolo risulti terminated dopo il click su Telegram
Dopo aver cliccato "✅ Fatto" su un task singolo in Telegram:
• Il task deve avere status = 'terminated' in ts_tasks
• La regola di notifica deve essere eliminata da cm_notification_rules
• Il row della coda deve avere status = 'completed'
⏳ da fare SQL Supabase 3° — test finale 
-- Ultimi task completati via Telegram (terminati di recente)
SELECT id, title, type, status, last_completed_date
FROM ts_tasks
WHERE status = 'terminated'
  AND last_completed_date >= now() - interval '30 minutes'
ORDER BY last_completed_date DESC
LIMIT 10;

-- Verifica che la regola di notifica sia stata eliminata
-- (sostituisci <TASK_ID> con l'UUID del task appena completato)
SELECT COUNT(*) AS regole_rimaste
FROM cm_notification_rules
WHERE entity_id = '<TASK_ID>'
  AND app = 'tasks';

-- Stato della queue per quel task
SELECT id, status, fire_at AT TIME ZONE 'Europe/Rome' AS fire_at_rome
FROM cm_notification_queue
WHERE entity_id = '<TASK_ID>'
ORDER BY fire_at DESC;

DB2 Guide v1.0.0 Guida: Stored Procedure KCBPANAG — z/OS
📋 Firma della Stored Procedure
DB2 z/OS — Signature
CALL KCBPANAG (
    PARM1,    -- IN  VARCHAR — parametro di input (chiave/codice)
    PARM2,    -- OUT VARCHAR — valore restituito 1
    PARM3,    -- OUT VARCHAR — valore restituito 2
    PARM4,    -- OUT VARCHAR — valore restituito 3
    PARM5     -- OUT VARCHAR — valore restituito 4
)
📊 Tabella Parametri
Parametro Direzione Tipo Descrizione
PARM1 IN VARCHAR Parametro di input — chiave o codice da passare alla proc
PARM2 OUT VARCHAR Primo valore restituito dalla stored procedure
PARM3 OUT VARCHAR Secondo valore restituito dalla stored procedure
PARM4 OUT VARCHAR Terzo valore restituito dalla stored procedure
PARM5 OUT VARCHAR Quarto valore restituito dalla stored procedure
🔢 Passi da seguire
  1. Prepara il REXX
    Crea un membro nel tuo dataset REXX con il codice di chiamata. Imposta il valore di PARM1 (il tuo parametro IN) e inizializza i parametri OUT a spazio vuoto.
  2. Connettiti al subsystem DB2
    Usa ADDRESS DSNREXX e il comando DSNREXX CONNECT DSN1 (sostituisci DSN1 con il nome del tuo subsystem DB2).
  3. Esegui la CALL
    Usa EXECSQL CALL KCBPANAG(:PARM1, :PARM2, :PARM3, :PARM4, :PARM5). I due punti (:) davanti al nome identificano le variabili host REXX.
  4. Controlla SQLCODE
    Verifica sempre la variabile SQLCODE dopo la CALL. 0 = successo, 100 = nessuna riga, negativo = errore.
  5. Leggi i parametri OUT
    Se SQLCODE = 0, le variabili PARM2, PARM3, PARM4, PARM5 contengono i valori restituiti dalla stored procedure. Usale direttamente nel codice REXX.
  6. Disconnetti e termina
    Chiama DSNREXX DISCONNECT per rilasciare la connessione DB2 prima di terminare il REXX.
📝 Codice REXX completo

Copia questo membro nel tuo dataset REXX (es. MY.REXX(KCBPANAG)). Modifica il nome del subsystem DB2 (DSN1) e il valore di PARM1_VAL.

REXX — MY.REXX(KCBPANAG)
/* REXX — Chiamata stored procedure KCBPANAG                    */
/* Parametri: PARM1 IN, PARM2/3/4/5 OUT                         */

ADDRESS DSNREXX

/* ------------------------------------------------------------ */
/* STEP 1 — Connessione al subsystem DB2                        */
/* ------------------------------------------------------------ */
"DSNREXX CONNECT DSN1"          /* <-- sostituire DSN1 con il tuo subsystem */
IF RC <> 0 THEN DO
    SAY 'ERRORE: Connessione DB2 fallita. RC=' RC
    EXIT 8
END

/* ------------------------------------------------------------ */
/* STEP 2 — Inizializzazione parametri                          */
/* ------------------------------------------------------------ */
PARM1 = 'VALORE_INPUT'            /* <-- IN:  imposta il tuo valore       */
PARM2 = ' '                        /* OUT: inizializzato a spazio vuoto    */
PARM3 = ' '                        /* OUT: inizializzato a spazio vuoto    */
PARM4 = ' '                        /* OUT: inizializzato a spazio vuoto    */
PARM5 = ' '                        /* OUT: inizializzato a spazio vuoto    */

/* ------------------------------------------------------------ */
/* STEP 3 — Chiamata alla stored procedure                      */
/* ------------------------------------------------------------ */
"EXECSQL CALL KCBPANAG(:PARM1, :PARM2, :PARM3, :PARM4, :PARM5)"

/* ------------------------------------------------------------ */
/* STEP 4 — Controllo SQLCODE e lettura parametri OUT           */
/* ------------------------------------------------------------ */
SELECT
    WHEN SQLCODE = 0 THEN DO
        SAY '=== KCBPANAG eseguita con successo ==='
        SAY 'PARM1 (IN)  :' PARM1
        SAY 'PARM2 (OUT) :' PARM2
        SAY 'PARM3 (OUT) :' PARM3
        SAY 'PARM4 (OUT) :' PARM4
        SAY 'PARM5 (OUT) :' PARM5
    END
    WHEN SQLCODE = 100 THEN DO
        SAY 'ATTENZIONE: Nessuna riga trovata (SQLCODE 100)'
    END
    OTHERWISE DO
        SAY '=== ERRORE SQL ==='
        SAY 'SQLCODE  =' SQLCODE
        SAY 'SQLERRM  =' SQLERRM
        SAY 'SQLSTATE =' SQLSTATE
        /* Disconnetti prima di uscire in errore */
        "DSNREXX DISCONNECT"
        EXIT 12
    END
END

/* ------------------------------------------------------------ */
/* STEP 5 — Disconnessione                                      */
/* ------------------------------------------------------------ */
"DSNREXX DISCONNECT"

EXIT 0
ℹ️ Note importanti
💡 Il prefisso : (due punti) davanti al nome variabile nella EXECSQL identifica una variabile host REXX. Senza i due punti DB2 non riconosce la variabile.
⚠️ Sostituire DSN1 con il nome reale del tuo subsystem DB2. Chiedere al DBA o verificare nel pannello ISPF DB2 Interactive.
✅ Non serve BIND né compilazione. Il REXX con DSNREXX è interpretato: modifica il membro e riesegui subito.
▶️ Come eseguire il REXX da ISPF
TSO Command Line
/* Da ISPF Option 6 (TSO Commands) */
EXEC 'MY.REXX(KCBPANAG)'

/* Oppure con EXEC e argomenti */
EXEC 'MY.REXX(KCBPANAG)' 'VALORE_INPUT'
⚙️ JCL per eseguire il REXX via batch

Usa questo JCL per lanciare il REXX in background (batch) senza dover aprire una sessione TSO interattiva.

JCL — MY.JCL(RUNKCBPA)
//*----------------------------------------------------------
//* JOB: Esecuzione REXX per chiamata KCBPANAG               
//*----------------------------------------------------------
//RUNKCBPA JOB (ACCT),'CALL KCBPANAG',
//         CLASS=A,MSGCLASS=X,
//         NOTIFY=&SYSUID
//*
//STEP1    EXEC PGM=IKJEFT01
//STEPLIB  DD DSN=DSN.SDSNEXIT,DISP=SHR      ← librerie DB2    
//         DD DSN=DSN.SDSNLOAD,DISP=SHR
//SYSTSPRT DD SYSOUT=*
//SYSPRINT DD SYSOUT=*
//SYSTSIN  DD *
  EXEC 'MY.REXX(KCBPANAG)'               ← dataset.membro REXX
/*
🔧 Cosa personalizzare nel JCL
CampoEsempioCosa mettere
(ACCT) (A123) Il tuo account/centro di costo
CLASS=A CLASS=B Job class del tuo sistema (chiedere al sysadmin)
DSN.SDSNEXIT DB2T.SDSNEXIT HLQ del dataset DB2 SDSNEXIT del tuo sito
DSN.SDSNLOAD DB2T.SDSNLOAD HLQ del dataset DB2 SDSNLOAD del tuo sito
MY.REXX PIPPO.REXX Il tuo dataset dove hai salvato il membro REXX
💡 Dopo il SUBMIT, controlla l'output in SDSF (opzione ST dal menu ISPF) filtrando per il tuo USERID. Il SAY del REXX apparirà nel SYSTSPRT (classe X = SYSOUT).
📤 Come fare il SUBMIT del JCL
  1. Apri il membro JCL nell'editor ISPF
    ISPF → Option 2 (Edit) → digita il dataset e il membro (es. MY.JCL(RUNKCBPA))
  2. Digita SUB sulla riga di comando
    Nella riga di comando in alto digita SUB e premi Invio. Il job viene sottomesso e riceverai un Job ID (es. JOB01234).
  3. Controlla l'output in SDSF
    ISPF → SDSF (opzione ST) → cerca il tuo job → premi S per aprirlo → controlla SYSTSPRT per il SAY del REXX e JESMSGLG per eventuali errori JCL.
  4. Verifica il Return Code
    RC=0000 = tutto OK. RC=0008 = errore connessione DB2. RC=0012 = errore SQL nella CALL.
🔴 SQLCODE comuni
SQLCODESignificatoSoluzione
0 Successo I parametri OUT contengono i valori restituiti dalla proc
+100 Nessuna riga trovata Il valore di PARM1 non ha trovato corrispondenza; verificare l'input
-204 Oggetto non trovato La stored procedure KCBPANAG non esiste nello schema indicato — verificare lo schema corretto con il DBA
-551 Mancanza di autorizzazione Il tuo USERID non ha il privilegio EXECUTE sulla stored procedure — chiedere al DBA il GRANT
-805 Package non trovato Il DBRM/Package della stored procedure non è stato fatto il BIND — contattare il DBA
-904 Risorsa non disponibile Il WLM address space della stored procedure non è attivo — contattare l'operatore z/OS
-911 Deadlock / timeout Conflitto di lock con un altro processo — riprovare dopo qualche secondo
-922 Autorizzazione fallita Token di connessione non valido — verificare le credenziali e la connessione al subsystem
🛠️ Errori frequenti in REXX
ProblemaCausaSoluzione
RC <> 0 su DSNREXX CONNECT Nome subsystem DB2 sbagliato Verificare il nome subsystem con il DBA (es. DB2P, DB2T, DBPX...)
REXX: variabile OUT resta vuota SQLCODE diverso da 0 Controllare sempre SQLCODE prima di usare i parametri OUT
IRXEXEC error / EXEC not found Dataset REXX non nel SYSEXEC Aggiungere nel JCL: //SYSEXEC DD DSN=MY.REXX,DISP=SHR
DSNREXX non riconosciuto DSN.SDSNLOAD non in STEPLIB Aggiungere DSN.SDSNLOAD e DSN.SDSNEXIT alla STEPLIB del JCL
